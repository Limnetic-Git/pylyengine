### PyLyEngine

# --- ЧТО ЗА ПРОЕКТ? ---
**Pylyengine** (пай-лай-энжин) - высокоуровневый движок с ECS (*Entity-Component System*), поддержкой камеры, динамического разрешения, покадровых анимаций, а также даже сетьевых модулей на **pysocknet** для создания многопользовательских игр!
Движок заточен в основном для 2D-topdown игр. Движок написан и использует **Python 3.X**, для отрисовки графики и воспроизведения звука - **raylib-python** (НЕ pyray!!!). Создаётся движок под проект **Limnetic**'а, - **Warants**.
Код полностью открыт, доступен к модификации и использованию в любых целях (в том числе и коммерческих).
Проект разрабатывает один из разработчиков **LyrarmaC#Engine**, а именно **LimneticGG** (*хех, это я :3*)

# --- ЧТО ОН МОЖЕТ? ---
Изначально инициализируется окно с начальным разрешением и нужными флагами. Далее инициализируется сцена, имеющая доступ к объекту окна. На сцену крепятся объекты. Сцены каждый кадр должна выполнять **.update()**, чтобы вызывать **.update()** для каждого объекта на сцене, у которых будет вызываться **.update()** для каждого модуля (компонента) привязанного к ним. В модулях находится основная логика, т.е. перемещение, отрисовка, физика и тп. Код для модулей и объектов желательно писать на основе шаблонов их объектов в **Modules/Module.py** и **Objects/Object.py** соответсвенно. Такая структура позволяет не прописывать одну и ту же логику для множества объектов, а просто привязать к ним нужные модули. Некоторые модули зависимы от других или от атрибутов объекта которые другие инициализируют, тогда вы увидите предупреждение или ошибку связанную с этим. Если вы не инициализируете нужные атрибуты для работы модуля (*например parent.speed для Movement*), то модуль сам инициализирует их для объекта на дефолтные значения (*для speed это будет 1, а например для x и y в Sprite2D, это 0 и 0*). Движок поддерживает слои (фактически просто сортировка объектов в очереди на **.update()** по **layer_id**), тайловые карты, камеру с перемещением и зумом, звук от объектов рядом через модуль **SoundEmitter**, покадровые анимации, оптимизации (*например объекты не попадающие на экран не отрисовываются*), информацию для тестирования (*FPS, RSS, DrawingCalls*), готовую модель перемещения и тайловые коллайдеры, а также базовый UI, с поддержкой настройки стилей в отдельном **JSON-файле** (*там есть всякие кнопки, чекбоксы, area, texture, ползунки, линия для ввода текста, текст и слот-лист*), базовый инвентарь, причем всё это масштабируется в размерах окна и считается в пропорциях, что упрощает разработку под устройства с разным разрешением, без костыля в виде фиксированного размера окна =)
